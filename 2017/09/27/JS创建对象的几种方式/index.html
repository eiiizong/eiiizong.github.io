JS创建对象的几种方式

发表于 2017-09-27 | 分类于 面向对象 |
字数统计: 4,832 字 | 阅读时长 ≈ 18 分钟

虽然Object构造函数或对象字面量都可以用创建单个对象。 但是这两个方式都有明显的缺点：使用同一个接口创建多个对象时，会产生很多的重复的代码。所以引入了 工厂模式 解决这种问题。

1 工厂模式

工厂模式是一种设计模式，这种模式抽象了创建具体对象的过程。因为ECMAScript中无法创建类，所以开发人员就发明了一种函数，用函数来封装以特定接口创建对象的细节。如下例所示：

函数 createPeople（）能够根据接收的参数来构建一个包含所有必要信息的people对象。可以无数次的调用这个函数，而每次它都返回一个包含两个属性和一个方法的对象。

在上面的代码中有这样一段代码 console.log(this);，而这个this在控制台输出的是什么呢？输出如下：

{
    name: "Eiiizong", 
    age: 22, 
    showInfo: ƒ
}

所以当前的 this 指向的是对象 people1。即和 console.log（people1） 的输出结果是一样的。

工厂模式虽然解决了创建多个相似对象的问题，但却没有解决对象识别问题（即怎么知道一个对象的类型？）。所以又出现了以下新模式。

2 构造函数模式

ECMAScript 中的构造函数可以用来创建特定类型的对象，比如Object和Array这样的原生构造函数，在运行时会自动出现在执行环境中。所以我们也可以创建自定义的构造函数，从而自定义该构造函数的属性和方法。将上面的实例重写如下：

在上面的代码中有同样有这么一段代码 console.log(this);，而这个this在控制台输出的又是什么呢？输出如下：

People{
    name: "Eiiizong", 
    age: 22, 
    showInfo: ƒ
}

所以当前的 this 指向的仍是对象 people1。即和 console.log（people1） 的输出结果还是一样的。与工厂模式中的相比多了一个变量名 People。

这个例子中，People( )函数取代了createPeople( )函数。People( ) 函数除了与 createPeople( )函数除了相同部分外还有不同部分，不同之处：

  • 没有显示地创建对象
  • 之间将属性和方法赋于 this 对象
  • 没有 return 语句

此外，People( ) 函数的函数名People的开头第一字母是大写的 P 。这种做法是一种习惯写法，主要是为了区别于ECMAScript中的其他函数，因为构造函数本身也是函数，只不过可以用来创建对象而已。

要创建People的新实例，必须使用 new 操作符。以这种方式调用构造函数实际上回经历以下四个步骤：

  • 1、创建一个新对象
  • 2、将构造函数的作用域赋给新对象 (因此 this 就指向了新对象)
  • 3、执行构造函数中的代码 (为这个新对象添加属性和方法)
  • 4、返回新对象

在前面的例子的最后 ，people1 保存着一个 People 的一个实例，这个people1对象有一个 constructor （构造函数）属性，该属性指向 People。例如：

//在上例末尾添加如下代码：
console.log(people1.constructor === People); //输出结果：true

对象的 constructor 属性最初是用来标识对象类型的。但是对于检测对象类型，还是 instanceof 操作符更可靠一些。我们在上例中创建的 people1 对象既是 Object 的实例，也是 People 的实例。这一点可以通过 instanceof 操作符得到验证：

//在上例末尾添加如下代码：
console.log(people1 instanceof People); //输出结果：true
console.log(people1 instanceof Object); //输出结果：true

创建自定义的构造函数意味着将来可以将它的实例 标识为一种特定的类型。这也是构造函数胜过工厂模式的地方。这个例子中之所以 people1 对象既是 Object 的实例，也是 People 的实例，是因为对象均继承与 Object。

注意：以这种方式定义的构造函数是定义在Global对象（在浏览器中是 Window 对象）中的。

构造函数与普通函数唯一的区别在与调用它们的方式不同。

上述问题就可以通过原型模式来解决~~

3 原型模式

在这里我们要知道：我们创建的每个函数都有一个 prototype(原型) 属性,这个属性其本质是一个 指针，指向一个对象。而这个对象包含可以由特定类型的所有实例共享的属性和方法。

按照字面意思来理解就是：prototype 就是调用构造函数而创建的实例的原型对象。

使用原型对象的好处是：所有对象实例共享它所包含的属性和方法。换句话说就是，不必在构造函数中定义对象的属性和方法，而是将这些属性和方法直接添加到原型对象中。如下所示：

在这个例子中，我们直接将showInfo方法和所有属性直接添加到了 People的prototype属性中，构造函数变成了空函数。即使这样我们也可以通过调用构造函数来创建新对象实例。而通过此构造函数创建的所有对象实例都拥有相同的属性和方法。它与构造函数模式不同的地方是：新对象实例的所有属性和方法是所有通过此构造函数创建的对象实例所共享的。换句话说就是 people1 和 people2 访问的都是同一组属性和同一个 showInfo方法。这点在上面代码中 console.log(people1.showInfo === people2.showInfo)返回值为 true 已得到验证。

3.1 理解原型对象

无论什么时候，我们创建一个函数，就会默认的给这个函数创建一个 prototype 属性，这个属性指向该函数的原型对象。同时在默认情况下，所有的原型对象还会自动获取一个constructor（构造函数）属性。这个属性也是一个指针，指向prototype属性所在函数的指针。

拿前面的例子来说，People.prototype.constructor 就指向 People 这个函数。通过这个People构造函数我们还可以继续为原型对象动态的添加其他的属性和方法。

创建一个自定义构造函数之后，其原型对象中只会默认包含一个constructor属性。至于其他方法就是继承于Object。当创建一个构造函数后，其内部默认包含一个prototype属性。利用构造函数创建一个实例后，该实例就会获取到这个属性，因为这个prototype属性本质是一个指针，所以该实例内部就包含了一个指针，该指针指向构造函数的原型对象。在ECMA-262 第5版中称这个指针叫[[prototype]],虽然在脚本中没有标准方式访问这个指针，但 Chrome 等浏览器在每个对象上都支持一个属性 _proto_。要明确一点的是这个连接存在于实例与原型对象之间，而不是实例与构造函数之间。

虽然在所有的实现中无法访问到[[prototype]]，但是可以通过 isPrototypeOf（）方法来确定对象之间是否存在这种关系。如果[[prototype]]指向 调用isPrototypeOf（）方法的对象，那么就返回 true。如下：

alert(People.prototype.isPrototypeOf(people1));//true

上面这段代码表示：people1这个对象中的[[prototype]]指针指向了People.prototype这个原型对象。所有返回 true。

ECMAScript 5 中新增了一个方法，叫 Object.getPrototypeOf（），在所有支持实现中，这个方法返回 [[prototype]] 的值。例如：

alert(Object.getPrototypeOf(people1) == People.prototype);//true
alert(Object.getPrototypeOf(people1).name);//Eiiizong

上面的第一段代码只是确定了Object.getPrototypeOf（）返回的对象就是这个对象的原型。第二段代码获取了原型对象中的 name 属性的值：Eiiizong。

3.1.1 对象属性和方法的搜索机制

每当代码读取到某个对象的某个属性时，就会进行一次搜索，目标是具有给定名字的属性。搜索过程如下：

  • 1、从对象实例本身开始搜索。如果有就返回，没有继续进行下一步搜索
  • 2、在对象实例本身包含的指针指向的原型对象中搜索。有就返回。

其方法的搜索机制和属性一样。
