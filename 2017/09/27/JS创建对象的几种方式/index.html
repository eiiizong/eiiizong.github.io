JS创建对象的几种方式

发表于 2017-09-27 | 分类于 面向对象 |

虽然Object构造函数或对象字面量都可以用创建单个对象。 但是这两个方式都有明显的缺点：使用同一个接口创建多个对象时，会产生很多的重复的代码。所以引入了 工厂模式 解决这种问题。

1 工厂模式

工厂模式是一种设计模式，这种模式抽象了创建具体对象的过程。因为ECMAScript中无法创建类，所以开发人员就发明了一种函数，用函数来封装以特定接口创建对象的细节。如下例所示：

函数 createPeople（）能够根据接收的参数来构建一个包含所有必要信息的people对象。可以无数次的调用这个函数，而每次它都返回一个包含两个属性和一个方法的对象。

在上面的代码中有这样一段代码 console.log(this);，而这个this在控制台输出的是什么呢？输出如下：

{
    name: "Eiiizong", 
    age: 22, 
    showInfo: ƒ
}

所以当前的 this 指向的是对象 people1。即和 console.log（people1） 的输出结果是一样的。

工厂模式虽然解决了创建多个相似对象的问题，但却没有解决对象识别问题（即怎么知道一个对象的类型？）。所以又出现了以下新模式。

2 构造函数模式

ECMAScript 中的构造函数可以用来创建特定类型的对象，比如Object和Array这样的原生构造函数，在运行时会自动出现在执行环境中。所以我们也可以创建自定义的构造函数，从而自定义该构造函数的属性和方法。将上面的实例重写如下：

在上面的代码中有同样有这么一段代码 console.log(this);，而这个this在控制台输出的又是什么呢？输出如下：

People{
    name: "Eiiizong", 
    age: 22, 
    showInfo: ƒ
}

所以当前的 this 指向的仍是对象 people1。即和 console.log（people1） 的输出结果还是一样的。与工厂模式中的相比多了一个变量名 People。

这个例子中，People( )函数取代了createPeople( )函数。People( ) 函数除了与 createPeople( )函数除了相同部分外还有不同部分，不同之处：

  • 没有显示地创建对象
  • 之间将属性和方法赋于 this 对象
  • 没有 return 语句

此外，People( ) 函数的函数名People的开头第一字母是大写的 P 。这种做法是一种习惯写法，主要是为了区别于ECMAScript中的其他函数，因为构造函数本身也是函数，只不过可以用来创建对象而已。

要创建People的新实例，必须使用 new 操作符。以这种方式调用构造函数实际上回经历以下四个步骤：

  • 1、创建一个新对象
  • 2、将构造函数的作用域赋给新对象 (因此 this 就指向了新对象)
  • 3、执行构造函数中的代码 (为这个新对象添加属性和方法)
  • 4、返回新对象

在前面的例子的最后 ，people1 保存着一个 People 的一个实例，这个people1对象有一个 constructor （构造函数）属性，该属性指向 People。例如：

//在上例末尾添加如下代码：
console.log(people1.constructor === People); //输出结果：true

对象的 constructor 属性最初是用来标识对象类型的。但是对于检测对象类型，还是 instanceof 操作符更可靠一些。我们在上例中创建的 people1 对象既是 Object 的实例，也是 People 的实例。这一点可以通过 instanceof 操作符得到验证：

//在上例末尾添加如下代码：
console.log(people1 instanceof People); //输出结果：true
console.log(people1 instanceof Object); //输出结果：true

创建自定义的构造函数意味着将来可以将它的实例 标识为一种特定的类型。这也是构造函数胜过工厂模式的地方。这个例子中之所以 people1 对象既是 Object 的实例，也是 People 的实例，是因为对象均继承与 Object。

注意：以这种方式定义的构造函数是定义在Global对象（在浏览器中是 Window 对象）中的。

2.1 构造函数即函数

构造函数与普通函数唯一的区别在与调用它们的方式不同。

构造函数也是函数，不存在什么特殊的语法，任何函数，只有通过 new 操作符来调用，那么它就可以作为构造函数。而任何函数，如果不通过 new 操作符来调用，那它就是普通函数。例如上例中的People（）函数可以通过下列方式调用：

在上例中当作构造函数调用的两段代码展示了构造函数的典型写法，即使用 new 操作符来创建一个新对象。 当作普通函数调用的两段代码展示了 不使用 new 操作符调用 People（）函数会出现什么结果：属性和方法都被添加到 window 对象了。所以我们可以通过 window 对象调用 People（）函数里的属性和方法。最后，也可以使用 call（）（或者apply（））在某个特殊的作用域中调用 People（）函数。这里是在对象 o 的作用域中调用，因此调用后， o 就拥有 People（）函数里的所有属性和方法。
