ECMAScript 基本概念

发表于 2017-08-21 | 分类于 JavaScript

1 语法

1.1 区分大小写

ECMAScript中的一切（包括 变量 ，函数名 ，操作符 ）都区分大小写。即意味着test与Test、tEst分别表示不同的变量。

1.2 标识符

所谓标识符就是指变量，函数，属性的名字，或者函数的参数。
标识符按照以下规则组合：

  • 1、第一个字符必须是字母，下划线（_）或一个美元符号（$）
  • 2、其他字符可以是字母，下划线，美元符号或者数字

注意：推荐使用驼峰大小写格式 例如: myCar

1.3 注释

1.4 严格模式

ECMAScript 5 引入了严格模式(strict mode)的概念。严格模式是为JavaScript定义了一种不同的解析与执行模型。

在严格模式下，ECMAScript 3 中的一些不确定的行为将得到处理，而且多某些不安全的操作也会抛出错误。

如果要在整个脚本中执行严格模式，在脚本顶部加上以下代码：

“use strict” ;

如果要指定某个函数执行严格模式，在函数内部顶部加上use strict;代码。

例如：

use strict; 这段代码看起来想字符串，而且也没有赋值给任何变量，但它其实是一个编译指示，用于告诉支持的JavaScript引擎切换到严格模式。这是为不破坏ECMAScript 3语法而特意选定的语法。
当然，在严格模式下，JavaScript的执行结果会有很大的不同。

注意：关键字也是语言保留的，不能用作标识符。

3 变量

ECMAScript 的变量是松散型的。所谓松散型就是可以用来保存任意类型的数据，即每个变量仅仅是一个用于保存值得站位符而已。定义变量是用var关键字。

var message；

这行代码定义了一个名为 message 的变量，该变量可以用来保存任意类型的值(像这样未经初始化的值会保存一个特殊值——undefined)。ECMAScript 也支持定义变量时直接初始化变量：

var message = “hello”;

上面的代码则是变量 message 一开始就保存了一个字符串”hello”。像这样初始化变量并不会把它标记为字符串类型，初始化的过程就是给变量赋一个值那么简单。因此，可以在修改变量值的时候同时修改值的类型，如下所示：

注意：使用 var 操作符定义的变量将成为该变量作用域的局部变量。也就是说如果你在某函数中使用 var 定义了一个变量，那么该变量在这个函数退出后将被销毁。
省略 var 操作符定义的变量将成为全局变量。但是不推荐这种做法，因为在局部作用域中定义全局变量很难维护，而且有意省略 var 操作符，相应变量不会马上就有定义而导致不必要的混乱。给未声明的变量赋值在严格模式下回导致抛出 ReferenceError（引用错误）错误。

可以使用一条语句定义多个变量，只有把每个变量（初始化或不初始化均可）用逗号分隔开即可。

4.3 Null 类型

Null 类型是第二个只有一个值的数据类型（第一个是undefined），这个特殊值是 null。

从逻辑角度看， null 表示一个 空对象指针，这也是我们使用 typeof 操作符检测 null 值时返回“object”的原因。

var names = null；
alert（typeof names）； //警告框显示：object

注意：上面的例子中，如果我们将变量名该为 name ,在谷歌浏览器上运行，那么警告框将会显示“ string”，而不是“object”，因为类似谷歌内核的浏览器将 name 视为关键字。

如果我们定义变量准备用来保存对象，那么我们最好将变量初始化为 null 而不是其他值。这样一来，只有直接检测 null 值就知道相应的变量是否已经保存了一个对象的引用。

注意：实际上，undefined 值是派生自 null 值的。因此 ECMA-262 规定对他们进行相等性测试，返回 true。

alert（null == undefined）； //返回 true

上面这里，位于 null 和 undefined 之间的相等操作符（==）总是返回 true ，不过要注意的是：这个操作符出于比较的目的会转换其操作数。

尽管 null 和 undefined 有这样的关系，但它们的用途完全不同。就像前面所讲的，无论什么情况下，都没有必要把一个变量的值初始化为undefined，但作为一个对象变量，但还没有真正保存对象，将其初始化为 null 是有必要的。这样做不仅可以体现 null 作为空对象指针的惯例，而且也有助于进一步区分 null 和 undefined。

1、十进制——最基本的数值字面量格式

2、八进制——以0开头，然后是八进制数值序列 (0~7)

3、十六进制——以0x开头，后跟十六进制数值序列 (0~9 及 A~F)

4.1.1 浮点数值

浮点数就是该数值中必须包含一个小数点，并且小数点后面必须至少有一位数字。小数点前面可以没有整数，但是不推荐这种写法。

因为保存浮点数值需要的内存空间是保存整数的两倍，所以ECMAScript 有机会就会把浮点数值转换为整数值。例如以下情况：

  • 小数点后面没有任何数字
  • 浮点数本身就是一个整数

对于极大或者极小的数值，可以用e(或者E)表示法（即科学计数法）表示浮点数值。e表示法表示的数值等于e前面的数乘以10的指数幂。如下所示：

var a = 7.23e5; // 等于 723000

注意：浮点数的的最高精度是 17 位小数（也就是小数点后面最多只能有 17 位数字）。但是其进行算术运算时的精度远远不如整数。例如：0.1加0.2的结果不是 0.3，而是 0.30000000000000004。

4.1.2 数值范围

由于内存限制，EXMAScript 不能保持世界上所有的数值。

  • EXMAScript 能够表示的最小数值保存在Number.MIN_VALUE 中
  • EXMAScript 能够表示的最大数值保存在Number.MAX_VALUE 中

如果某次计算超出了javascript数值范围，那么这个数值将会自动转换成特殊的 Infinity值。如果为正数，则转换为Infinity(正无穷)；如果为负数，则转换为-Infinity(负无穷)。

注意：如果某次计算返回了正或负的Infinity值，该值将无法参与下一次的计算。

4.3.3 NaN

NaN,即非数值(Not a Number) 是一个特殊的数值。这个数值表示：本来要返回数值的操作数未返回数值的情况。比如在其他编程语言中，任何数值除以 0 都会导致错误，从而停止执行代码。但在 ECMAScript 中，任何数值除以 0 会返回 NaN ，因此不会影响其他代码的执行。

NaN 本身有两个非同寻常的特点：

  • 任何涉及 NaN 的操作(例如：NaN / 10 )都会返回 NaN。
  • NaN 与任何值都不相等，包括其本身。

针对这两个特点，ECMAScript 定义了 isNaN() 函数。这个函数接受一个参数，该参数可以是任何类型，而函数会帮助我们确定这个参数是否“不是数值”， isNaN()在接受一个值之后，会尝试将这个值转换为数值。任何不能被转换为数值的值都会导致这个函数返回 true，能则为 false。如下例所示：

有 3 个函数可以把非数值转换为数值：

  • Number( ) —— 可用于任何数据类型
  • parseInt( ) —— 用于把字符串转换为整数数值

4.5.4.1 Number( )函数

  • 如果是 Boolean 值， true 和 false 分别转换为 1和 0

  • 如果是 null 值，返回 0
  • 如果是 undefined ，返回 NaN
  • 如果是 字符串 ，遵循以下规则：
    • 如果字符串中只包含数字，则将其转换对应为十进制数。

    • 如果字符串中包含有效的十六进制格式，则将其转换为相同大小的十进制数值。
    • 如果字符串为空，则将其转换位0。
    • 如果字符串中包含除上述以外的字符，则将其转换为NaN。

4.6 String 类型

String 类型用于表示由零或多个16位 Unicode 字符组成的字符序列，即字符串。字符串可以由双引号(“)和单引号(‘)表示。不过以双引号开头的字符串必须以双引号结束，单引号同理。

4.7 Object 类型

ECMAScript 中的对象其实就是一组数据和功能的集合。对象可以通过执行new 操作符，后跟要创建的对象类型的名称来创建。而创建 Object 类型的实例并为其添加属性和方法，就可以创建自定义对象，如下：

var cars = new Object( );

在ECMAScript 中，如果不给构造函数传递参数，则可以省略后面那对圆括号（不推荐这种做法）。

var cars = new Object;  //有效，但不推荐

仅仅创建Object的实例并没什么用处，但要理解一个重要的思想：即在ECMAScript 中Object 类型是它所有实例的基础。换句话说，Object 类型所具有的任何属性和方法也同样存在于更具体的对象中。

Object 的每个实例都具有下列的属性和方法：

  • Constructor: 保存着用于当前对象的函数。对于前面的例子而言，构造函数（constructor）就是Object( )。
  • hasOwnProperty(propertyName): 用于检查给定的属性在当前对象实例中（而不是在实例的的原型中）是否存在。其中，作为参数的属性名（propertyName）必须以字符串的形式指定（例如：cars.hasOwnProperty(“Name”)）。
  • isPrototypeOf(Object): 用于检查当前的对象是否是当前对象的原型。
  • propertyIsEnumerable(propertyName): 用于检查给定的属性是否能够使用 for-in 语句来枚举。与hasOwnProperty( )方法一样，作为参数的属性名必须以字符串的形式指定。
  • toLocaleString(): 返回对象的字符串表示，该字符串与执行环境的地区对应。
  • toString(): 返回对象的字符串表示。
  • valueOf(): 返回对象的字符串、数值或布尔值表示。通常与toString()方法返回的值相同。

注意：从技术角度讲，ECMA-262中的对象不一定适用于 javascript 中的其他对象。浏览器环境中的对象，比如DOM和BOM中的对象，都属于宿主对象，因为它们是由宿主实现和定义的。ECMA-262 不负责定义宿主对象，因此宿主对象可能会也可能不会继承Object。

5 操作符

ECMA-262描述了一组用于操作数据值的操作符，包括算术操作符（比如加号和减号）、位操作符、关系操作符和相等操作符。

只能操作一个值的操作符叫做一元操作符。

5.2 位操作符

位操作符用于在最基本的层次上，即按内存中表示数值的位来操作数值。在 ECMAScript 中所有的数值都以 IEEE-754 64位格式存储。但位操作符并不直接操作 64 位的值，而是先将 64 位的值转换为 32 位的值，然后在进行操作，最后将结果换回 64 位的值。

对于有符号的整数。32位中的前31位表示整数的值。第 32 位（从即左向右数的第一位）用于表示数值的符号：即 0 表示正数，1 表示负数。这个表示符号的位，叫做符号位。

正数以纯二进制格式存储，31位中每一位都表示2的幂。从右到左依次为2的 0、1、2、3···次幂。没有用到的位以 0 填充。例如：数值 18 的二进制表示为：00000000000000000000000000010010，当然可以简写为10010。

负数同样二进制码存储，但使用的格式是二进制补码。通过以下 3 步得到一个数值的补码：

  • 首先求这个数值的绝对值的二进制码
  • 求二进制的反码，即0换成1，,1换成0
  • 最后在得到的二进制反码加 1

例如：以 -18 为例。

  • 首先通过绝对值求得 18，在计算 18 的二进制码。即得： 00000000000000000000000000010010
  • 第二步，求其反码，即将0和1互换。即得： 11111111111111111111111111101101
  • 最后二进制反码加1，即得：
    11111111111111111111111111101110

这样我们就得到了 -18 的二进制码：11111111111111111111111111101110。要注意的是，在处理有符号的整数时，是不能访问 31 位的。

ECMAScript 会尽力想我们隐藏所有这些信息。例如我们在以一个二进制字符串的形式输出一个负数时，我们看到的只是这个负数绝对值的二进制码前面加了一个负号。例如：

注意: 默认情况下，ECMAScript 中所有整数都是有符号整数。不过，也存在无符号整数。对于无符号整数来说，第 32 位不再表示符号，因为无符号整数只能是正数。而且无符号整数更大，因为多出了一位不再表示符号，可以用来表示数值。

ECMAScript 中当对数值应用为操作符时，后台会发生如下转换过程：

  • 64 位的数值被转换为 32 位数值
  • 然后，执行位操作
  • 最后，将 32 位数值换回 64 位数值

这样表面看起来就好像是在操作 32 位数值，但这个转换过程也导致了一个严重的副效应，即在对特殊的 NaN 和 Infinity 值应用位操作时，这两个值会被当成 0 来处理。如果对非数值用于位操作符，会首先使用 Number() 函数将该值转换为一个数值（自动完成），然后在进行位操作。得到的结果将是一个数值。

5.3 布尔操作符

布尔操作符一个有三个：非(NOT)、 与(AND) 和 或(OR)。

5.3.1 逻辑非

逻辑非操作符由一个感叹号(!)表示。可以应用于 ECMAScript 中的任何值。遵循以下规则：

  • 如果操作数是一个对象，返回 false
  • 如果操作数是一个空字符串，返回 true
  • 如果操作数是一个非空字符串，返回 false
  • 如果操作数是数值 0，返回 true
  • 如果操作数是任意非 0 数值(包括 Infinity)，返回 false
  • 如果操作数是null，返回 true
  • 如果操作数是NaN，返回 true
  • 如果操作数是undefined，返回 true

逻辑与操作符由两个和号(&&)表示，有两个操作数。其真值表如下：

第一个操作数 第二个操作数 结果

  • 如果第一个操作数是对象，则返回第二个操作数
  • 如果第二个操作数是对象，则只有在第一个操作数为true的情况下才返回该对象

  • 如果有一个操作数是NaN，则返回NaN
