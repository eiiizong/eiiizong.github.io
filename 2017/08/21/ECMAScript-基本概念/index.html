ECMAScript 基本概念

发表于 2017-08-21 | 分类于 JavaScript

1 语法

1.1 区分大小写

ECMAScript中的一切（包括 变量 ，函数名 ，操作符 ）都区分大小写。即意味着test与Test、tEst分别表示不同的变量。

1.2 标识符

所谓标识符就是指变量，函数，属性的名字，或者函数的参数。
标识符按照以下规则组合：

  • 1、第一个字符必须是字母，下划线（_）或一个美元符号（$）
  • 2、其他字符可以是字母，下划线，美元符号或者数字

注意：推荐使用驼峰大小写格式 例如: myCar

1.3 注释

1.4 严格模式

ECMAScript 5 引入了严格模式(strict mode)的概念。严格模式是为JavaScript定义了一种不同的解析与执行模型。

在严格模式下，ECMAScript 3 中的一些不确定的行为将得到处理，而且多某些不安全的操作也会抛出错误。

如果要在整个脚本中执行严格模式，在脚本顶部加上以下代码：

“use strict” ;

如果要指定某个函数执行严格模式，在函数内部顶部加上use strict;代码。

例如：

use strict; 这段代码看起来想字符串，而且也没有赋值给任何变量，但它其实是一个编译指示，用于告诉支持的JavaScript引擎切换到严格模式。这是为不破坏ECMAScript 3语法而特意选定的语法。
当然，在严格模式下，JavaScript的执行结果会有很大的不同。

1.5 语句

ECMAScript中的语句以一个分号（;）结束；如果省略分号，则由解析器确定语句的结尾。虽然有效，但是不推荐省略分号的写法。

我们可以采用C风格的语法把多条语句组合到一个代码块中。即代码块以左花括号( { )开始，以右花括号( } )结尾:

例如：

if(test) {
test = ture;
alert(test);
}

注意：虽然条件控制语句（例如if语句）只在执行多条语句的情况下要求使用代码块，但最佳实践是始终在控制语句中使用代码块——即使代码块中只有一条语句。这样能降低修改代码时出错的几率。

2 关键字和保留字

ECMA-262描述了一组具有特定用途的关键字，这些关键字可以用于表示控制语句的开始和结束。或者用于执行特定的操作符等。

注意：关键字也是语言保留的，不能用作标识符。

ECMAScript的全部关键字如下所示(带*号上标的是第5版新增的关键字)：

  • case
  • catch

  • default

  • new

  • try
  • typeof
  • var
  • void
  • while
  • with

ECMA-262同时还描述了另一组不能用作标识符的保留字。尽管保留字在这门语言中还没有任何特定用途，但他们有可能在将来被用作关键字。

以下是ECMA-262第3版定义的全部保留字：

  • abstract
  • boolean
  • byte
  • char
  • class
  • const
  • debugger
  • double
  • enum
  • export

  • final
  • float
  • goto
  • implements
  • import
  • int
  • interface
  • long
  • native
  • package

  • public
  • short
  • static
  • super
  • synchronized
  • throws
  • transient
  • volatile

第5版把在非严格模式下运行时的保留字缩减为下列这些：

  • class
  • const
  • enum
  • export
  • extends
  • import
  • super

在严格模式下，第5版还对以下保留字施加了限制(带*号上标的是第5版新增的保留字)：

  • implements
  • interface
  • let *
  • package
  • private
  • protected
  • public
  • static
  • yield *

注意：在实现ECMAScript 3的JavaScript引擎中使用关键字作标识符，会导致“Identifier Expected(缺少标识符)”的错误。而使用保留字作标识符可能会也可能不会导致相同的错误，具体取决于特定的引擎。

一般来说，最好都不要使用关键字和保留字作为标识符和属性名，以便于与将来的ECMAScript版本兼容。

除了上面列出的保留字和关键字，ECMA-262第5版对evel和arguments还施加了限制。在严格模式下这两个名字不能作为标识符和属性名，否则会抛出错误。

3 变量

ECMAScript 的变量是松散型的。所谓松散型就是可以用来保存任意类型的数据，即每个变量仅仅是一个用于保存值得站位符而已。定义变量是用var关键字。

var message；

这行代码定义了一个名为 message 的变量，该变量可以用来保存任意类型的值(像这样未经初始化的值会保存一个特殊值——undefined)。ECMAScript 也支持定义变量时直接初始化变量：

var message = “hello”;

上面的代码则是变量 message 一开始就保存了一个字符串”hello”。像这样初始化变量并不会把它标记为字符串类型，初始化的过程就是给变量赋一个值那么简单。因此，可以在修改变量值的时候同时修改值的类型，如下所示：

var message = “hello”;

message = 10; //有效，但不推荐

虽然我们不建议修改所保存值的类型，但这种操作在 ECMAScript 中完全有效。

注意：使用 var 操作符定义的变量将成为该变量作用域的局部变量。也就是说如果你在某函数中使用 var 定义了一个变量，那么该变量在这个函数退出后将被销毁。
省略 var 操作符定义的变量将成为全局变量。但是不推荐这种做法，因为在局部作用域中定义全局变量很难维护，而且有意省略 var 操作符，相应变量不会马上就有定义而导致不必要的混乱。给未声明的变量赋值在严格模式下回导致抛出 ReferenceError（引用错误）错误。

可以使用一条语句定义多个变量，只有把每个变量（初始化或不初始化均可）用逗号分隔开即可。

var mane = “zxm”,

age = 29;

4 数据类型

ECMAScript 中有5种简单数据类型：Undefined、Null、Boolean、Number 和 String。还有一种复杂类型—— Object , Object 本质上是由一组无序的名值对组成的。

4.1 typeof 操作符

鉴于 ECMAScript 是松散类型的，因此需要一种手段来检测给定的变量的数据类型—— typeof 就负责提供这方面信息的操作符。对一个值使用 typeof 操作符可能返回下列某个字符串：

“undefined”——————如果这个值未定义

“boolean”——————如果这个值布尔值

“number”——————如果这个值是数值

“string”——————如果这个值是字符串

“object”——————如果这个值是对象或null

“function”——————如果这个值是函数

例如：

var name = Eiiizong;  
alert(typeof name);       // 警告框显示：string
alert(typeof (name));    // 警告框显示：string
alert(typeof age);      // 产生错误，警告框显示：undefined
alert(typeof "age");   // 警告框显示：string
alert(typeof 22);     // 警告框显示：number
alert(typeof null);  // 警告框显示：object

这个例子说明 typeof 操作符的操作数可以是变量（name）,也可以是数值字面量（22）。 注意：typeof 是操作符而不是函数，因此第三行句子中的圆括号尽管可以使用，但不是必需的。

调用 typeof null 会返回 “object”,因为特殊值 null 被认为是一个空的对象引用。

注意：从技术角度将，函数在 ECMAScript 中是对象，不是一种数据类型。但是函数也确实有一些特殊的属性，因此通过 typeof 操作符来区分函数和其他对象是有必要的。

4.2 Undefined 类型

Undefined 类型只有一个值，即特殊的 undefined 。在使用 var 声明变量但未对其初始化时，这个变量的值就是 undefined，例如：

var name；
alert（name == undefined）； //警告框显示：true

这个例子只声明了 name 变量，但未对其进行初始化，比较这个变量与 undefined 字面量，结果两者是相等的。这个例子与下面的例子是等价的：

var name = undefined；
alert（name == undefined）； //警告框显示：true

这个例子使用 undefined 值初始化了变量 name 。但我们没有必要这么做，因为未经初始化的变量会默认取得 undefined 值。但是，包含 undefined 值的变量与没有定义的值还是不一样的。

var name；//这个变量声明过后，默认取得 undefined 值
//下面这个变量没有声明
// var age
alert（name）； //警告框显示：undefined
alert（age）； //产生错误

运行以上代码，在第一个警告框弹出 undefined 后，第二个警告框——由于传递给alert（）函数的是未定义的变量 age —— 导致一个错误，从而终止代码运行。

注意：定义未声明的变量只能执行一项操作，即使用 typeof 操作符检测其数据类型

解释：对于未初始化和未声明的变量进行 typeof 操作符都返回了 undefined 值。这个结果有其逻辑上的合理性。因为虽然这两种变量从技术角度看有本质区别，但实际上无论对哪种变量都不可能执行真正的操作。

基于以上，如果我们显示地初始化变量，那么当我们使用操作符 typeof 返回“undefined”值的时候，我们就知道被检测的变量是还没有被声明，而不是尚未初始化。

4.3 Null 类型

Null 类型是第二个只有一个值的数据类型（第一个是undefined），这个特殊值是 null。

从逻辑角度看， null 表示一个 空对象指针，这也是我们使用 typeof 操作符检测 null 值时返回“object”的原因。

var names = null；
alert（typeof names）； //警告框显示：object

注意：上面的例子中，如果我们将变量名该为 name ,在谷歌浏览器上运行，那么警告框将会显示“ string”，而不是“object”，因为类似谷歌内核的浏览器将 name 视为关键字。

如果我们定义变量准备用来保存对象，那么我们最好将变量初始化为 null 而不是其他值。这样一来，只有直接检测 null 值就知道相应的变量是否已经保存了一个对象的引用。

注意：实际上，undefined 值是派生自 null 值的。因此 ECMA-262 规定对他们进行相等性测试，返回 true。

alert（null == undefined）； //返回 true

上面这里，位于 null 和 undefined 之间的相等操作符（==）总是返回 true ，不过要注意的是：这个操作符出于比较的目的会转换其操作数。

尽管 null 和 undefined 有这样的关系，但它们的用途完全不同。就像前面所讲的，无论什么情况下，都没有必要把一个变量的值初始化为undefined，但作为一个对象变量，但还没有真正保存对象，将其初始化为 null 是有必要的。这样做不仅可以体现 null 作为空对象指针的惯例，而且也有助于进一步区分 null 和 undefined。

4.4 Boolean 类型

Boolean 类型只有两个字面值：true 和 false。这两个值与数字值不是一回事，因此 true 不一定等于1，false 不一定等于0。

虽然Boolean 类型只有两个字面值， 但 ECMAScript 中所有类型的值都有与这两个Boolean 值等价的值。要想将一个值转换为其对应的Boolean值，可以调用转型函数Boolean(),如下所示：

var message = "Hello World!";
var messageAsBoolean = Boolean(message);
alert(messageAsBoolean);              //警告框显示：true
