Promise 对象

发表于 2019-03-08 | 分类于 ES6 |
字数统计: 1,938 字 | 阅读时长 ≈ 8 分钟

Promise 是异步编程的一种解决方案。比传统的 回调函数和事件 更加合理更强大。 ES6 将其写入标准，统一用法，原生提供 Promise
对象。

1. Promise 的含义

Promise，简单说就是一个容器，里面保存着某个未来才会结束的事件（该事件通常是一个异步操作）结果。

从语法上说，Promise 是一个对象，从它可以获取异步操作的消息。

Promise 对象有以下两个特点：

  • 对象的状态不受外界的影响。
  • 一旦状态改变，就不会再变，任何时候都可以得到这个结果。

Promise 对象代表一个异步操作，有三种状态：

  • pending 进行中
  • fulfilled 已成功
  • rejected 已失败

只有异步操作的结果可以决定 Promise 当前是属于哪一种状态，其他任何操作都无法改变这个状态。也就是其特点一。

Promise 对象的状态改变只有两种可能：

  • 从 pending 变为 fulfilled
  • 从 pending 变为 rejected

只要上面的两种可能有一种发生，状态就凝固，不会再改变了，这时就称为 resolved（已定型）。如果状态已经改变了，再对 Promise 对象添加回调函数，也会立即得到这个结果。也就是其特点二。

这与事件（Event）完全不同，事件的特点是，如果你错过了它，再去监听，是得不到结果的。

Promise 对象的缺点：

  • 无法取消 Promise: 一旦新建就会立即执行，无法中途取消。
  • 如果不设置回调函数，Promise 内部抛出错误，不会反应到外部
  • 当处于 pending 状态时，无法得知目前进展到哪一阶段（刚刚开始还是即将完成）

ECMAScript 6 简介

字数统计: 283 字 | 阅读时长 ≈ 1 分钟

ECMAScript 6.0 （以下简称 ES6）是 JavaScript 语言的下一代标准，在 2015 年 6 月 正式发布。

1. ECMAScript 和 JavaScript 的关系

1996 年 11 月， JavaScript 的制造者 Netscape 公司，希望 JavaScript 这种语言能够成为国际标准，于是将其提交给标准化组织 ECMA。

次年，ECMA 发布 262 号标准文件（ECMA-262）的第一版（这个版本就是 1.0 版），规定了浏览器脚本语言的标准。并将这种语言称为 ECMAScript。

但是不叫 JavaScript 有两个原因：

  • 商标： JavaScript 本身也已经被 Netscape公司注册为商标。所以只有 Netscape 公司可以合法地使用 JavaScript 这个名字。
  • 体现这门语言的制定者是 ECMA，不是 Netscape。

ECMAScript 和 JavaScript 的关系：前者是后者的规格， 后者是前者的一种实现。
